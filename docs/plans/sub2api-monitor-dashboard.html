Sub2API 监控看板

一个全新的 Next.js + React 监控大屏，用永久 Admin API Key 从 Sub2API 管理接口拉取数据，持续展示 token 使用量、请求量和用户排行。页面不展示邮箱，不展示费用，默认无人值守自动刷新和轮播。

Next.js App Router React 大屏 UI x-api-key 服务端接入 今日 / 7 天 / 30 天

目标

首版只做一个酷炫、稳定、信息密度合适的排行榜。页面默认打开即运行，主要面向监控屏，不要求操作者频繁点击。

Top N默认展示前 20 名，人数更多时自动分页。
0 邮箱每行展示用户名，不展示用户邮箱。
0 费用只展示 token、请求数和趋势，不展示成本。

接口事实

已确认 Sub2API 管理接口不必依赖页面运行时登录。管理路由支持 x-api-key 认证，适合把永久 Admin API Key 放在新应用的服务端环境变量里。

  • Admin API Key 管理接口：GET /api/v1/admin/settings/admin-api-key、POST /api/v1/admin/settings/admin-api-key/regenerate、DELETE /api/v1/admin/settings/admin-api-key。
  • 当前实例查询结果为 exists: false，上线前需要先在 Sub2API 管理端生成一次 key。
  • 排行榜汇总接口：GET /api/v1/admin/dashboard/users-ranking。
  • 用户趋势接口：GET /api/v1/admin/dashboard/users-trend。
  • users-ranking 适合排序，返回结构为 data.ranking，但不稳定提供用户名。
  • users-trend 适合趋势和用户名补全，返回结构为 data.trend，每行可包含 username。
  • 已通过生成的 Admin API Key 验证：直接使用 x-api-key 可以访问排行榜和趋势接口，不依赖登录态。
BFF 层按 user_id 合并两个接口：排序用 ranking，展示名优先用 trend 的 username，缺失时显示 用户 #<短 user_id>，不退回邮箱。

产品范围

包含

  • 一个监控大屏首页。
  • 一个服务端 API 聚合层，统一调用 Sub2API 管理接口。
  • 今日、近 7 天、近 30 天三个时间窗口。
  • Top N 排行、自动分页、趋势 sparkline、刷新状态和最近更新时间。
  • 排名变化、数字滚动、趋势线绘制、行高亮、周期切换过渡。

不包含

  • 多页面后台系统、应用内登录系统、CSV 导出、告警规则。
  • 费用展示、邮箱展示、多实例聚合。
  • 对 Sub2API 的写操作。生成 Admin API Key 是一次性运维动作，不在监控页面里做。

动效原则

  • 排名变化平滑移动，升降名次用短暂颜色和箭头提示。
  • 新进入榜单淡入并短暂高亮，离开当前页淡出后移除。
  • token 和请求数使用 600 到 900ms 的数字滚动。
  • 趋势线刷新后从左到右重绘。
  • 自动分页和周期轮播使用平滑过渡，避免瞬间跳屏。
  • 尊重 prefers-reduced-motion。

定时刷新规则

  • 首次打开页面立即请求一次数据。
  • 数据刷新间隔默认 60 秒，通过 LEADERBOARD_REFRESH_SECONDS 配置。
  • 周期自动轮播默认 20 秒，通过 LEADERBOARD_PERIOD_ROTATE_SECONDS 配置。
  • 数据刷新和周期轮播相互独立。
  • 每次刷新都重新计算时间范围，默认时区 Asia/Shanghai。
  • 今日窗口为当天 00:00:00 到当前时间。
  • 近 7 天窗口为包含今天在内的最近 7 个自然日。
  • 近 30 天窗口为包含今天在内的最近 30 个自然日。
  • 如果上一轮请求还没结束，跳过本次刷新，不并发叠加。
  • 请求失败时保留上一份成功快照，顶部显示失败状态和失败次数。
  • 页面不可见时可以降低刷新频率到 5 分钟；重新可见后立即刷新一次。
  • 自动分页默认每 12 秒翻一页。

数据模型

type LeaderboardPeriod = "today" | "week" | "month";

type LeaderboardRow = {
  rank: number;
  userId: string;
  displayName: string;
  tokens: number;
  requests: number;
  trend: Array<{
    bucket: string;
    tokens: number;
    requests: number;
  }>;
  rankDelta?: number;
};

type LeaderboardSnapshot = {
  period: LeaderboardPeriod;
  generatedAt: string;
  timezone: string;
  totals: {
    tokens: number;
    requests: number;
    activeUsers: number;
  };
  rows: LeaderboardRow[];
};

displayName 优先使用 trend 数据里的 username；为空时显示 用户 #<userId 前 6 位>；禁止使用邮箱兜底。

技术方案

  • Next.js App Router。
  • React Client Component 负责动画和定时刷新。
  • Route Handler 负责服务端聚合和密钥保护。
  • TypeScript。
  • Tailwind CSS 或项目内轻量 CSS 变量。
  • 图表使用轻量 SVG sparkline，不引入重型图表库。
Context7 查阅的 Next.js 文档确认：App Router 的 Route Handler 可以用标准 GET(request: Request) 处理 API；服务端代码可通过 process.env 读取环境变量；频繁变化数据应使用 fetch(..., { cache: "no-store" }) 或等价方式避免缓存。

实施单元

1. 初始化全新 Next.js 项目

建立 package.json、next.config.ts、tsconfig.json、app/layout.tsx、app/page.tsx、app/globals.css。清理旧部署项目文件，仅保留 docs/，并添加只属于本应用的配置。

测试：lint、typecheck、首页空数据状态。

2. Sub2API 服务端客户端

实现 lib/env.ts、lib/sub2api/client.ts、lib/sub2api/time-windows.ts、lib/sub2api/leaderboard.ts、types/leaderboard.ts。封装 x-api-key 请求、时间窗口、ranking + trend 合并和展示名兜底。

测试：时间窗口、username 缺失兜底、输出不含 email、请求失败错误。

3. Next.js 聚合 API

实现 GET /api/leaderboard，支持 period 和 limit，使用 no-store，返回统一快照。

测试：非法 period、key 缺失、Sub2API 认证失败、成功快照结构。

4. 大屏排行榜 UI

实现排行榜主屏、行组件、顶部指标、周期提示和 SVG sparkline。行内只展示排名、用户名、token、请求数和趋势图。

测试：不出现 cost、不出现 email、sparkline 点数正确、1440x900 和 1920x1080 截图检查。

5. 自动刷新、轮播和动画

实现首次加载、60 秒刷新、20 秒周期轮播、自动分页、请求并发保护、排名/数字/趋势动画和 prefers-reduced-motion。

测试：fake timers 覆盖刷新、轮播、并发保护和失败保留旧数据。

验收标准

  • 打开首页无需点击即可看到排行榜。
  • 页面默认自动在今日、近 7 天、近 30 天之间轮播。
  • 数据默认每 60 秒刷新一次。
  • 每行只显示排名、用户名、token、请求数和趋势图。
  • 页面任何位置都不展示用户邮箱。
  • 页面任何位置都不展示费用。
  • Admin API Key 只存在服务端环境变量中，不进入浏览器 bundle 或接口响应。
  • 今日趋势按小时展示，7 天和 30 天趋势按天展示。
  • 超过可见行数时自动分页或自动滚动，不需要鼠标操作。
  • Sub2API 短暂不可用时保留最后一次成功数据。

风险与处理

  • users-ranking 不直接返回 username：通过 users-trend 合并，缺失时显示匿名用户编号。
  • 当前实例还没有 Admin API Key：上线前需要在 Sub2API 管理端生成一次，并写入 .env.local。
  • 监控屏长时间运行可能内存增长：只保留当前和上一份快照用于动画对比。
  • 数据刷新和动画可能互相干扰：刷新只替换快照，动画由 React 状态对比驱动。
  • 小项目容易过度设计：首版不引入数据库、不做登录、不做复杂权限、不做重型图表平台。
后续可选：增加最近 24 小时滚动窗口、模型维度或 API key 维度排行榜、异常峰值提示、只读分享页、部署脚本和健康检查。
